# Type of Licences
## 1. MIT

- **Type:** Permissive (Open Source).

- **Golden Rule:** You can do almost anything as long as you include the original copyright notice.

- **Modifications:** You are not required to release your modified source code.

- **Commercial Use:** Fully allowed; you can even bundle it into a paid product.

## 2. GPL (General Public License)

- **Type:** Strong Copyleft (Open Source).

- **Golden Rule:** If you modify and distribute the software, your code must also be licensed under the GPL.

- **"Viral" Effect:** If your project uses a piece of GPL code, the entire project usually becomes GPL.

- **Commercial Use:** Allowed, but you must provide the source code to whoever uses or buys the software.

## 3. LGPL (Lesser GPL)

- **Type:** Weak Copyleft (Open Source).

- **Golden Rule:** Designed primarily for code libraries.

- **Proprietary Use:** You can link a proprietary program to an LGPL library without having to open your program's source code.

- **Modifications:** If you modify the library itself, you must share those specific changes.

## 4. Apache 2.0

- **Type:** Permissive with Legal Focus (Open Source).

- **Golden Rule:** Similar to MIT, but includes explicit terms regarding patents.

- **Protection:** Contributors automatically grant you the right to use any patents involved in the code.

- **Modifications:** You can keep your code private, but you must include a list of major changes made to the original files.

## 5. Proprietary

- **Type:** Closed License (Commercial).

- **Golden Rule:** The author or company retains all rights; you only "rent" or buy the right to use it.

- **Code Access:** Usually impossible (the source code is a trade secret).

- **Restrictions:** Copying, modifying, or redistributing the software without an express contract or payment is prohibited.
