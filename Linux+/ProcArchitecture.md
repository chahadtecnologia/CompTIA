# Proc Architecture

## 1. Intel/AMD x86 and x86_64

- **Definition:** The standard architecture for almost all PCs and laptops since the 1980s.

- **x86:** Refers to the 32-bit version (older). It is limited to addressing a maximum of 4GB of RAM.

- **x86_64:** The 64-bit extension. This is the modern standard, allowing for massive amounts of memory.

- **Highlight:** Based on the CISC (Complex Instruction Set Computer) model; very powerful but generally consumes more power and generates more heat.

## 2. AMD64

- **Definition:** This is essentially the same as x86_64.

- **History:** While Intel created the 32-bit (x86) standard, AMD was the one that successfully designed the 64-bit extension we use today.

- **Highlight:** This is why in many systems (like Linux installers), you will see the architecture labeled as "amd64" even if you are using an Intel processor.

## 3. ARM (aarch64)

- **Definition:** An architecture focused on energy efficiency. It dominates the mobile market (iPhone, Android) and newer Macs (M1/M2/M3 chips).

- **aarch64:** The technical name for the 64-bit ARM instruction set.

- **Highlight:** Based on the RISC (Reduced Instruction Set Computer) model. It performs simpler instructions very quickly, resulting in excellent battery life.

## 4. IBM Z (s390x)

- **Definition:** The architecture for IBM Mainframes. You won't find these in stores; they are used by massive banks and government agencies.

- **s390x:** The technical name for the 64-bit version of this architecture.

- **Highlight:** Designed for extreme availability and processing trillions of financial transactions simultaneously with high security.

## 5. RISC-V

- **Definition:** The "new kid on the block." It is a completely Open Source architecture.

- **Freedom:** Unlike ARM or Intel, any company can manufacture a RISC-V chip without paying royalties or licensing fees.

- **Highlight:** It is considered the future of modular computing, growing rapidly in IoT (Internet of Things) devices and experimental servers.
